# FinOps Maturity Model

**Level:** Advanced  
**Framework:** FinOps Foundation  
**Last Updated:** January 2025

---

## Overview

The FinOps Maturity Model helps organizations assess their FinOps maturity and plan improvements.

---

## Maturity Levels

### Crawl

**Characteristics:**
- Basic FinOps practices
- Manual processes
- Limited visibility
- Reactive approach

**Capabilities:**
- Basic cost visibility
- Manual cost allocation
- Periodic optimization
- Basic reporting

### Walk

**Characteristics:**
- Good FinOps practices
- Automated processes
- Good visibility
- Proactive approach

**Capabilities:**
- Detailed cost allocation
- Automated allocation
- Regular optimization
- Good reporting
- Performance tracking

### Run

**Characteristics:**
- Excellent FinOps practices
- Fully automated
- Real-time visibility
- Continuous optimization

**Capabilities:**
- Real-time cost understanding
- Fully automated allocation
- Continuous optimization
- Real-time reporting
- Real-time benchmarking
- Real-time decisions

---

## Maturity Assessment

### Assess Each Capability

Assess maturity for each of the six capabilities:
1. Understanding Cloud Usage and Cost
2. Performance Tracking and Benchmarking
3. Real-Time Decision Making
4. Cloud Rate Optimization
5. Cloud Usage Optimization
6. Organizational Alignment

### Overall Maturity

Overall maturity is typically the average of capability maturities.

---

## Maturity Progression

### Typical Progression

1. **Start:** Crawl level across all capabilities
2. **Progress:** Move capabilities to Walk
3. **Advanced:** Move capabilities to Run
4. **Continuous:** Maintain and improve

### Not All Capabilities at Same Level

Organizations typically have different maturity levels for different capabilities.

---

## Key Takeaways

1. **Three levels:** Crawl, Walk, Run
2. **Assess capabilities:** Assess each capability
3. **Progressive:** Progress over time
4. **Not uniform:** Different capabilities at different levels
5. **Continuous:** Always improving
