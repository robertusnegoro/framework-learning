## Overview

ISO/IEC 19770 is an international standard for Software Asset Management (SAM). It provides a framework for managing software assets effectively and ensuring compliance.

---

## ISO 19770 Standard

### Standard Structure

ISO 19770 consists of multiple parts:

- **ISO 19770-1:** Processes and tiered assessment
- **ISO 19770-2:** Software identification tags (SWID)
- **ISO 19770-3:** Entitlement schema
- **ISO 19770-5:** Overview and vocabulary
- **ISO 19770-8:** Guidelines for mapping to ISO/IEC 19770-1

### ISO 19770-1: Processes

**Purpose:** Defines SAM processes and tiered assessment model.

**Key Components:**
- SAM processes
- Process categories
- Tiered assessment
- Maturity levels

**Process Categories:**
- Management and control
- Lifecycle
- Operations
- Verification and compliance

### ISO 19770-2: SWID Tags

**Purpose:** Defines software identification tags.

**Key Components:**
- SWID tag structure
- Tag types
- Tag creation
- Tag usage

**Tag Types:**
- Primary tags
- Patch tags
- Corpus tags
- Product tags

### ISO 19770-3: Entitlement Schema

**Purpose:** Defines entitlement schema for software licenses.

**Key Components:**
- Entitlement structure
- License information
- Usage rights
- Compliance data

## Software Asset Management (SAM)

### SAM Definition

**Software Asset Management (SAM)** is the practice of managing software assets throughout their lifecycle to maximize value, minimize costs, and ensure compliance.

### SAM Scope

**Includes:**
- Software licenses
- Software installations
- License compliance
- Software optimization
- Vendor management

### SAM Objectives

**Primary Objectives:**
- License compliance
- Cost optimization
- Risk reduction
- Value maximization

## SWID Tags

### SWID Tag Overview

**Software Identification Tag (SWID Tag)** is a standardized tag that identifies software products and their relationships.

### SWID Tag Structure

**Components:**
- Software identity
- Product information
- Version information
- Relationships
- Metadata

### SWID Tag Types

#### 1. Primary Tags

**Purpose:** Identify installed software products.

**Characteristics:**
- Product identification
- Version information
- Installation details
- Relationships

**Usage:**
- Software identification
- Installation tracking
- Compliance management

#### 2. Patch Tags

**Purpose:** Identify software patches and updates.

**Characteristics:**
- Patch identification
- Parent product
- Patch details
- Relationships

**Usage:**
- Patch tracking
- Update management
- Compliance tracking

#### 3. Corpus Tags

**Purpose:** Identify software collections.

**Characteristics:**
- Collection identification
- Member products
- Collection details
- Relationships

**Usage:**
- Collection management
- Suite tracking
- Compliance management

#### 4. Product Tags

**Purpose:** Identify software products (not installed).

**Characteristics:**
- Product identification
- Product details
- Availability
- Relationships

**Usage:**
- Product catalog
- License management
- Procurement
