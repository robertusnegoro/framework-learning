## Overview

Assessment discovery is the process of gathering information about the current state of ITAM in an organization. It provides the foundation for planning ITAM implementation and improvements.

---

## Discovery Process

### Phase 1: Planning

#### 1. Define Scope

**Scope Elements:**
- **Geographic Scope:** Locations to assess
- **Asset Scope:** Types of assets to assess
- **Process Scope:** Processes to assess
- **Technology Scope:** Systems to assess
- **Time Scope:** Assessment period

**Scope Definition:**
- Identify assessment boundaries
- Define inclusion/exclusion criteria
- Set priorities
- Document scope

#### 2. Identify Stakeholders

**Stakeholder Categories:**
- **Executive Sponsors:** Leadership support
- **ITAM Team:** ITAM practitioners
- **IT Operations:** IT operations staff
- **Finance:** Financial management
- **Procurement:** Procurement team
- **End Users:** Asset users
- **Vendors:** External vendors

**Stakeholder Mapping:**
- Identify all stakeholders
- Map relationships
- Assess influence
- Plan engagement

#### 3. Prepare Discovery Plan

**Plan Components:**
- Discovery objectives
- Scope definition
- Timeline
- Resources
- Methods and tools
- Deliverables

**Plan Approval:**
- Review plan
- Obtain approvals
- Communicate plan
- Set expectations

---

### Phase 2: Data Collection

#### 1. Documentation Review

**Documents to Review:**
- **Policies:** ITAM policies and procedures
- **Processes:** Process documentation
- **Reports:** Existing reports and metrics
- **Contracts:** Vendor contracts and agreements
- **Financial Records:** Budget and spending records
- **Audit Reports:** Previous audit findings
- **Organizational Charts:** Structure and roles

**Review Process:**
- Collect documents
- Review content
- Extract relevant information
- Document findings
- Identify gaps

**Documentation Checklist:**
- [ ] ITAM policies
- [ ] Process documentation
- [ ] Organizational charts
- [ ] Contracts and agreements
- [ ] Financial records
- [ ] Audit reports
- [ ] Previous assessments

#### 2. Stakeholder Interviews

**Interview Planning:**
- Identify interviewees
- Prepare questions
- Schedule interviews
- Conduct interviews
- Document findings

**Interview Questions (Sample):**

**For ITAM Team:**
- What ITAM processes are currently in place?
- What tools are used for ITAM?
- What are the main challenges?
- What data is tracked?
- How accurate is the data?

**For IT Operations:**
- How are assets discovered?
- How are assets tracked?
- What systems are used?
- What are the pain points?
- What improvements are needed?

**For Finance:**
- How are IT costs tracked?
- What financial data is available?
- How are budgets managed?
- What reporting is done?
- What improvements are needed?

**For Procurement:**
- What is the procurement process?
- How are vendors managed?
- What contracts exist?
- How are purchases tracked?
- What improvements are needed?

**Interview Best Practices:**
- Prepare questions in advance
- Create comfortable environment
- Listen actively
- Take notes
- Follow up as needed

**Data Quality:**

- What issues exist?

**Technical Assessment Methods:**
- System reviews
- Tool evaluations
- Data quality analysis
- Integration assessment
- Gap analysis

#### 4. Process Assessment

**Processes to Assess:**
- Discovery processes

**Assessment Areas:**

### Phase 3: Data Analysis

#### 1. Data Consolidation

**Consolidation Activities:**
- Aggregate data from all sources
- Organize data by category
- Normalize data formats
- Remove duplicates
- Validate data

**Data Categories:**
- Process data
- Technology data
- Data quality data
- Organizational data
- Financial data

#### 2. Gap Analysis

**Gap Analysis Process:**
- Define target state
- Compare current vs. target
- Identify gaps
- Prioritize gaps
- Document gaps

**Gap Categories:**
- **Process Gaps:** Missing or ineffective processes
- **Technology Gaps:** Missing or inadequate tools
- **Data Gaps:** Data quality or completeness issues
- **Organizational Gaps:** Structure or capability issues
- **Governance Gaps:** Policy or compliance issues

**Gap Prioritization:**
- **Critical:** High impact, high urgency
- **Important:** High impact, medium urgency
- **Standard:** Medium impact, medium urgency
- **Low:** Low impact, low urgency

#### 3. Maturity Assessment

**Maturity Assessment:**
- Assess current maturity level
- Evaluate capabilities
- Identify maturity gaps
- Determine target maturity
- Plan maturity improvement

**Maturity Levels:**
- Level 0: Incomplete
- Level 1: Initial
- Level 2: Managed
- Level 3: Established
- Level 4: Predictable
- Level 5: Optimizing

## Discovery Methods and Tools

### Method 1: Questionnaires

**Purpose:** Collect structured information from stakeholders.

**Advantages:**
- Scalable
- Consistent data
- Easy to analyze
- Time-efficient

**Disadvantages:**
- Limited depth
- Response bias
- Low response rates

**Best Practices:**
- Keep questions clear
- Use multiple choice where possible
- Include open-ended questions
- Follow up as needed

### Method 2: Interviews

**Purpose:** Collect detailed information from key stakeholders.

**Advantages:**
- Rich information
- Clarification possible
- Relationship building
- Detailed insights

**Disadvantages:**
- Time-consuming
- Requires skilled interviewers
- May be biased
- Difficult to scale

**Best Practices:**
- Prepare questions
- Create comfortable environment
- Listen actively
- Take detailed notes
- Follow up

### Method 3: Workshops

**Purpose:** Collaborative discovery sessions.

**Advantages:**
- Multiple perspectives
- Collaborative insights
- Relationship building
- Comprehensive coverage

**Disadvantages:**
- Requires coordination
- May be dominated by few
- Time-consuming
- Requires facilitation

**Best Practices:**
- Clear objectives
- Good facilitation
- Diverse participation
- Document outcomes

### Method 4: Technical Assessment

**Purpose:** Assess technical capabilities and systems.

**Methods:**
- System reviews
- Tool evaluations
- Data quality analysis
- Integration assessment
- Performance testing

**Tools:**
- System documentation
- Tool configurations
- Data analysis tools
- Testing tools

### Method 5: Process Observation

**Purpose:** Observe processes in action.

**Advantages:**
- Real-world insights
- Identifies issues
- Validates documentation
- Practical understanding

**Disadvantages:**
- Time-consuming
- May affect behavior
- Requires access
- Limited scope
